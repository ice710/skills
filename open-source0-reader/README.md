# Open Source Project Reader Skill

一个用于系统学习开源项目的 Claude Code Skill。它会带你从项目速览、核心机制理解、目录地图、运行验证、源码追踪一路推进到可保存的网页学习报告。

## 这个 Skill 能做什么

- 拆解开源项目的定位、技术栈、核心功能和学习价值
- 用通俗语言解释项目解决的问题和内部工作方式
- 梳理目录结构、入口文件、核心模块、配置、示例、测试和文档
- 选择一条关键链路追踪源码调用过程
- 生成架构图、流程图、时序图辅助理解
- 标记暂时讲不清的地方，并给出补洞计划
- 最终沉淀成一份可浏览、可打印、可导出的 HTML 学习报告

## 适合什么时候用

当你想：

- 学习一个 GitHub 开源项目
- 看懂一个框架或工具的源码
- 拆解项目架构和核心流程
- 为技术分享、源码阅读、项目调研准备材料
- 生成一份结构化、图文结合的项目学习报告

## 输入信息

推荐提供以下信息：

```text
项目地址：
我的技术背景：
学习目标：
时间预算：
重点想看的部分：
是否需要网页报告：
是否需要图形化输出：
```

如果只提供项目地址，Skill 会默认按“理解项目原理、架构和核心源码”的目标推进。

## 工作流程

1. **项目速览**  
   先理解项目是什么、解决什么问题、适合谁用、为什么值得学。

2. **通俗解释**  
   用简单类比解释项目核心机制，再映射回技术概念。

3. **跑通项目**  
   梳理环境依赖、安装步骤、最小示例、关键配置和常见报错。

4. **建立项目地图**  
   拆解目录结构，定位入口文件、核心目录、测试、示例和文档。

5. **架构图**  
   用 SVG / Excalidraw 风格图说明模块边界、依赖关系和数据流向。

6. **核心流程追踪**  
   选择一条最关键的路径，例如 API 请求处理、CLI 命令执行、插件加载或任务调度。

7. **流程图 / 时序图**  
   用图形说明输入如何经过解析、调度、执行并产生输出，以及对象之间的调用顺序。

8. **不懂点与补洞**  
   主动标记讲不清的地方，并回到文档、源码、测试或 issues 中查证。

9. **网页学习报告**  
   生成包含文字、图形、源码追踪、不懂点和最终学习笔记的 HTML 报告。

## 输出内容

最终报告通常包含：

- 项目速览
- 一句话解释
- 通俗类比
- 核心概念解释
- 项目目录地图
- 架构图与说明
- 核心流程图与说明
- 关键调用链时序图与说明
- 源码追踪表
- 关键文件阅读卡片
- 不懂点列表
- 补洞计划
- 最终学习笔记
- 下一步学习计划
- 可贡献方向
- SVG / Excalidraw 图形资产区

## 文件说明

```text
feynman-open-source/
├── SKILL.md
├── open_source_reader.md
├── feynman_open_source_project_report_template.html
└── README.md
```

- `SKILL.md`：Claude Code Skill 主文件。
- `open_source_reader.md`：原始学习流程说明与设计文档。
- `feynman_open_source_project_report_template.html`：网页报告模板，支持本地保存、导出 HTML、打印 / 存 PDF。
- `README.md`：当前说明文档。

## 使用示例

```text
帮我用这个 skill 学习 https://github.com/xxx/yyy
我的目标是理解它的架构、核心源码和插件机制，最后生成一份网页学习报告。
```

也可以更具体：

```text
帮我拆解这个开源项目，重点看 CLI 命令是如何被解析和执行的。
需要架构图、动态流程图、时序图和源码追踪表。
最后生成 HTML 报告。
```

## 图形化能力

这个 Skill 优先使用可嵌入网页的 SVG 图形，适合生成：

- 项目整体架构图
- 核心调用流程图
- 模块交互时序图
- 数据流 / 控制流说明图

图形的目标不是炫技，而是让读者快速看懂：

- 项目的边界在哪里
- 核心模块在哪里
- 外部输入从哪里进入
- 数据如何在模块之间流动
- 哪些地方是扩展点或容易出错的节点

## 网页报告模板能力

`feynman_open_source_project_report_template.html` 支持：

- 浏览器直接打开
- 表单内容保存到 `localStorage`
- 导出为单文件 HTML
- 打印或另存为 PDF
- 上传 SVG / PNG / JPG 图形资产
- 左侧章节导航
- 架构图、流程图、时序图、源码追踪和最终笔记分区

## 学习完成标准

使用这个 Skill 学完一个项目后，你应该能回答：

- 我能不能用一句话解释这个项目？
- 我能不能不用术语解释它的核心机制？
- 我能不能画出它的核心模块关系？
- 我能不能追踪一次完整调用链？
- 我能不能解释关键函数为什么这样写？
- 我能不能自己写一个最小 demo？
- 我能不能指出项目里一个可以改进的地方？
- 我能不能给别人讲 10 分钟？

## 设计原则

- 先建立全局地图，再深入源码
- 围绕问题读代码，不逐行硬读
- 先讲清楚“为什么”，再讲“怎么做”
- 每个核心模块都要能用一句话解释
- 每张图都必须有配套说明
- 不懂的地方要显式记录，并形成补洞计划
- 最终产物要能长期保存和复习
